# Precheck — observable surface area

Load this reference at pipeline step 4.2. Apply it to every resolved file
before any rubric runs. Passing files proceed to step 4.3; failing files
are logged with a skip reason for the footer (see
[output-schema](./output-schema.md)).

## Pass criteria

A file passes the precheck if it has at least one of:

1. **A public/exported boundary** — HTTP handler, RPC server method, CLI entry, queue consumer, scheduled job.
2. **An external I/O call** — network, DB, filesystem, subprocess, or channel-spanning operation.
3. **Non-trivial error edges** — at least two distinct error returns or raise/throw points.

### Criterion 1 — public boundary

A file passes when it exposes a function or method called directly by a
transport, scheduler, or broker (HTTP router, gRPC server, CLI framework,
queue consumer, cron). Package-internal helpers do not count.

**Correct (passes):**

```go
func (s *Server) HandleIngest(w http.ResponseWriter, r *http.Request) error {
    // registered with http.Handle — transport boundary
}
```

**Wrong (does not pass):**

```go
func validatePayload(b []byte) error {
    // called only by HandleIngest; never exposed to a transport
}
```

Package-internal helper with no transport exposure — fails criterion 1.

### Criterion 2 — external I/O call

A file passes when it contains at least one call that crosses a process
boundary: network, DB driver, filesystem, subprocess, or cross-thread
channel. Pure in-memory computation does not qualify.

**Correct (passes):**

```python
def fetch_config(bucket: str, key: str) -> dict:
    obj = s3_client.get_object(Bucket=bucket, Key=key)
    return json.load(obj["Body"])
```

**Wrong (does not pass):**

```python
def merge_configs(base: dict, override: dict) -> dict:
    return {**base, **override}
```

Pure in-memory dict merge with no I/O — fails criterion 2.

### Criterion 3 — non-trivial error edges

A file passes when it contains at least two distinct error return or
raise/throw sites — independent error paths from different operations or
conditions. A single error return does not qualify.

**Correct (passes):**

```go
func LoadConfig(path string) (*Config, error) {
    data, err := os.ReadFile(path)
    if err != nil {
        return nil, fmt.Errorf("read: %w", err) // error edge 1
    }
    var cfg Config
    if err := json.Unmarshal(data, &cfg); err != nil {
        return nil, fmt.Errorf("parse: %w", err) // error edge 2
    }
    return &cfg, nil
}
```

**Wrong (does not pass):**

```go
func Validate(x string) error {
    if x == "" {
        return errors.New("empty")
    }
    return nil
}
```

Exactly one error edge — fails criterion 3, which requires at least two
distinct error returns. (Use criterion 1 or 2 to evaluate this file
instead; if none apply, assign a skip reason.)

## Skip reasons

When a file fails all three pass criteria, assign exactly one reason from
this fixed set: `generated`, `no-io`, `test-fixture`, `pure-data`, `vendored`.
Do not invent new reasons; if none fits, default to `no-io`.

When multiple skip reasons could apply (e.g., a vendored protobuf file is
both `vendored` and `generated`), check them in the order listed above and
assign the first that applies. This makes assignment deterministic.

### `generated`

Trigger: a machine-generated header at the top of the file (within the first
five non-blank lines). Recognized patterns: `// Code generated by ... DO NOT
EDIT` (Go), `# Generated by ...` (Python), `// @generated` (TS/JS).

**Correct (reason applies):**

```go
// Code generated by protoc-gen-go. DO NOT EDIT.
// source: api/ingest.proto

package api
```

**Wrong (does NOT trigger this reason):**

```python
"""Generates user reports nightly."""

def build_report(...):
    ...
```

Hand-written module whose docstring merely *mentions* generation — no
machine-generated header. Do not assign `generated`.

### `no-io`

Trigger: the file contains application logic (conditionals, loops, functions)
but passes none of the three criteria above — utility functions, formatters,
and validators with no external I/O and no transport boundary.

**Correct (reason applies):**

```python
def slugify(text: str) -> str:
    return re.sub(r"[^a-z0-9]+", "-", text.lower()).strip("-")
```

**Wrong (does NOT trigger this reason):**

```python
def fetch_status(url: str) -> int:
    return requests.get(url).status_code
```

Has external I/O — passes criterion 2 of the pass-criteria; do not skip
with `no-io`.

### `test-fixture`

Trigger: path contains `testdata/` or `__fixtures__/`, or filename ends in
`.golden`.

**Correct (reason applies):**

```text
pkg/ingest/testdata/valid_payload.json
```

**Wrong (does NOT trigger this reason):**

```go
// pkg/handler_test.go
func TestHandleIngest(t *testing.T) {
    got := HandleIngest(req)
    if got.Status != 200 {
        t.Fatalf("want 200, got %d", got.Status)
    }
}
```

Real test code, not a fixture — must NOT be skipped under `test-fixture`.
See the `What NOT to skip` section.

### `pure-data`

Trigger: the file declares only types, structs, dataclasses, constants, or
enums, with no methods beyond trivial accessors. Any method performing
computation, I/O, or error handling disqualifies the file.

**Correct (reason applies):**

```python
from dataclasses import dataclass

@dataclass
class IngestRecord:
    id: str
    body: bytes
    source: str
```

**Wrong (does NOT trigger this reason):**

```go
type User struct {
    Email string
}

func (u *User) Validate() error {
    if !strings.Contains(u.Email, "@") {
        return errors.New("invalid email")
    }
    return nil
}
```

Methods carry logic — do not skip with `pure-data`.

### `vendored`

Trigger: file path falls under `vendor/`, `node_modules/`, or a directory
listed in `.gitignore` as a dependency cache. Path-based; no example needed.

## What NOT to skip

- Test code (`*_test.go`, `tests/test_*.py`) is NOT auto-skipped. Tests can
  contain o11y anti-patterns the user wants flagged. Apply the three pass
  criteria normally; skip only when all three fail.
- Files of any size are NOT auto-skipped on length alone. Length is not a
  signal of observable surface area.

## Action checklist

At pipeline step 4.2:

1. For each resolved file, check pass criteria 1–3 in order; first match wins.
2. If none match, assign a skip reason from the fixed set above (default to
   `no-io` if none fits).
3. Record the file path and reason in the precheck rejection log (consumed by
   the footer at step 4.5 — see [output-schema](./output-schema.md)).
4. Pass surviving files to step 4.3.

**Audit:** every file resolved at step 4.1 must appear exactly once in
either the surviving set or the rejection log.
